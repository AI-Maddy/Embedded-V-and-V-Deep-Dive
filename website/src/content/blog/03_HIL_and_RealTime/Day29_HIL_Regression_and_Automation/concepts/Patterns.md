# 🧭 Patterns

-   **Define Invariants**: Establish invariants first, then derive
    measurable checks to ensure consistency. This helps in maintaining a
    stable testing environment.
-   **Explicit Interface Contracts**: Keep interface contracts explicit
    (units, ranges, rates) to avoid ambiguity. This is essential for
    ensuring that all components interact as expected.
-   **Evidence Mapping**: Map each concept to at least one evidence
    artifact to ensure traceability and accountability. This is crucial
    for compliance with standards like ISO 26262 and IEC 62304.
